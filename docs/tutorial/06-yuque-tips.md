---

## 语雀写作规范

### 1. 用「纯文本模式」写作

语雀编辑器右下角切换：

- **富文本模式** ❌ → 导出会带一堆 `<font>` 标签
- **Markdown 模式** ✅ → 导出就是干净 Markdown

### 2. 标题层级要规范

```
# 一级标题（文章标题，只用一次）
## 二级标题（章节）
### 三级标题（小节）
```

不手动改字号、颜色，用标题样式就行。

### 3. 图片处理

语雀的图片会上传到 `cdn.nlark.com`，导出后会有防盗链问题。

**建议做法**：
- 写笔记时正常插图
- 导出后统一用 `download-images.ps1` 脚本下载到本地

### 4. 避免在正文中写 `<!--`

`<!--` 会被 Vue 当作 HTML 注释开始标记，导致构建报错。

```markdown
<!-- 错误写法 -->
这句提到了<!--注释-->相关内容

<!-- 正确写法 -->
这句提到了反引号包起来的 `<!--` 注释相关内容
```

### 5. 表格不要合并单元格

语雀的合并单元格导出后会变成混乱 HTML，尽量用简单表格。

### 6. 代码块标注语言

```markdown
<!-- 正确 -->
```js
console.log('hello')
```


<!-- 错误 -->
```
console.log('hello')
```
```

不需要高亮的用 `plain` 或 `text`。

### 7. 不要手动改字号颜色

语雀里选中文字改颜色/字号，导出就是 `<font color="red" style="...">`。用 Markdown 语法替代：

| 想要的效果 | 用 Markdown |
|-----------|-------------|
| 加粗 | `**文字**` |
| 斜体 | `*文字*` |
| 行内代码 | `` `代码` `` |
| 引用 | `> 引用内容` |

### 总结

语雀写作时记住一条原则：**只用标题、列表、代码块、加粗、链接这几种基础格式**，不碰富文本工具栏里的颜色/字号/缩进——导出到 VitePress 基本不用改。
