### 1.1 理解一下NMOS和PMOS

S和D是要接VCC或者GND。G要接一个控制电路，来控制开关的打开和闭合,从而让S和D导通。未来真正的电路是在S和D之间流动的。

### 1.2 通用推挽输出

![](/images/img-81.png)


### 1.3 通用开漏输出
去掉P-MOS,即P-MOS永远关闭
![](/images/img-83.png)

>向输出数据寄存器写1,N-MOS断开,输出高阻抗;
向输出数据寄存器写0,N-MOS闭合,输出低电平

>高阻抗即MCU和外部针脚彻底断开,相当于开路。或者说N-MOS的D漏级和外部引脚互相悬空了。

![](/images/img-84.png)

但是我们不太想要有这个高阻态。如何改变呢？因为MCU内部结构都已经设计好,所以比如我想拉取一个电压,必须在MCU外部,且根据自己的实际需求决定。

```
单片机内部,不是只有MCU内部有VDD供电,其他MCU外部的设备,或者MCU内部的其他设备,也可能会有自己的供电源提供高电压。类比:自己家里(单片机)有很多家用电器(外设控制器和外设),它们都需要电。因此,在开漏模式下,要输出高电平,就必须在引脚位置设置上拉电阻,来提供高电压。
```
>写0时，NMOS闭合时，虽然有上拉电阻，但是连接到地上了，达到0V
写1时，NMOS断开，有上拉电阻，因此是高电平。当然外拉电阻实在MCU外部的，需要外加的。

为什么开漏输出高阻抗后,引脚就是高电平?
![](/images/img-85.png)

上图是欧姆定律:V=I*R。
> 一个电路回路中,电压会被各个元器件承接,电阻值越高,承压越高。

引脚的本质就是一根导线,因此当输出寄存器写1时即N-MOS断开时可看成:![](/images/img-86.png)

如果整个电路通路，从 VDD 到 GND  
• 整个电路的电流  I = 3.3V / (R_PULL+ R1 + R2) ,因为 R2 接近于⽆穷⼤，所以 I≈0  
• ⽽引脚位置的电压 V引脚 = 3.3V - 上拉电阻处的承压 = 3.3V - I * R_PULL  
• 因为 I≈0 ， 所以 V引脚 = 3.3V - I * R_PULL  =  3.3V - 0 * R_PULL = 3.3V  
• 所以，引脚处电压就是 3.3V ，那么输出就是⾼电平

可是电路不是通路啊,其实是开路了,按理说开路就没有电流,由V=IR,那么整个电路结构中电压应该是0啊,包括引脚处。那如何正确理解呢？

先来看一个网上的问题:一块裸的电池,没有接入到任何电路中,其实就是自身开路,正极有电压吗?  有！

:![](/images/img-87.png)
>电压的本质是两端(正负等)电势差，开路情况也可能存在;就好像⽔⻰头，不打开，也有⽔压⼀样 

:![](/images/img-88.png)

:![](/images/img-89.png)
```
以 R1 为例： 
• R1 处的电压,不是 R1_left  或  R1_right 处的电压,而是 R1_right - R1_left 的电压差,表⽰总电压是 10V,⾃⼰承担了 2V 的压力,这个叫做分压 
• ⽽ R1_right 真实的电压是 10V,R1_left 真实的电压是 10V-承压V1 = 10V-2V = 8V。所以,如果我要知道 R2_right 处的电压(注意,不是 R2 承担的电压),怎么算?不就是 10V-承压V1 = 8V 嘛！即  R1_left处电压 = R2_right处电压 (导线不承担电压) 
```
>欧姆定律计算的电压：是对应设备在整个电路中承担的电压差，即经过该设备，电压会下降多少。

核心就是区分承压和单点的概念。单点的电压是去掉承压的。比如上图R1,R1_right是10V,R1_left就是8V,因为要去掉承压2V。欧姆定律算的是承压情况。

:![](/images/img-90.png)
这就是开漏输出⾼阻抗之后,引脚就是⾼电平的原因。

###  2.1 介绍一下TTL施密特触发器
施密特触发器就是进行信息处理⼀个硬件单元。它的核⼼作用是: 抗噪整形：把抖动、毛刺、斜坡等不可靠的输⼊波形整形成规则的方波。所谓的方波,在硬件上是方波,在软件就是0/1,本质是数字信息。
:![](/images/img-96.png)

当MCU某一个引脚要进行输入的时候,因为外部因素的干扰,输入信息,不一定是规整的输入信息。比如:按键抖动、旋钮转动不均匀。
:![](/images/img-97.png)

如果没有 TTL 施密特触发器,那么输⼊信息就可能在01之间反复横跳。


如果存在 TTL 施密特触发器,当这类信息输⼊到引脚,传递到 TTL 施密特触发器,输⼊波形就会被整形成规则的⽅波,这个⼯作是 TTL 施密特触发器的核⼼作⽤。 

:![](/images/img-98.png)
:![](/images/img-99.png)

如何理解:迟滞区域的理解:输入抖动在迟滞区内,输出状态保持上次状态不变,避免误跳变。

### 2.3 根本问题
1.上拉输⼊的时候,输⼊低电平,上拉电阻不会把电平拉高吗？

2.下拉输⼊的时候,输⼊⾼电平,下拉电阻不会把电平拉低吗？  

这两个问题,其实原理上是同⼀种问题。我能理解是输⼊和上下拉电阻对抗的问题。
```
无论上拉电阻,还是下拉电阻,阻值在单片机内部,都属于高阻值器件,阻值一般都高达30kΩ~50kΩ。

欧姆定律V=I*R,在计算电压角度,计算的是对应设备在整个电路中承担的电压差,即经过该设备,电压会下降多少。
```
回答问题1:
:![](/images/img-104.png)

输入低电平是一种说法,在单片机内部,即可以理解成为就是通过接地实现引脚输入低电平的,那么其实内部上拉电阻和外部接地,就形成了一个通路。

引脚就是导线,电阻忽略不计,那么这个通路中唯一的一个需要承压的器件就是上拉电阻!那么上拉电阻两端的压差就是VDD,等效电路如下:
:![](/images/img-105.png)

所以上拉输入的时候,输入低电平,上拉电阻不会把电平拉高。因为形成通路以后,位置2电压就是0V,低电平,不受上拉电阻的影响。

回答问题2:
:![](/images/img-106.png)

输入高电平是一种说法,在单片机内部,即可以理解成就是通过接电源VDD实现引脚输入高电平的,那么其实内部下拉电阻和外部电源VDD,就形成了一个通路。

引脚就是导线,电阻忽略不计,那么这个通路中唯一的一个需要承压的器件就是下拉电阻!那么上拉电阻两端的压差就是VDD,等效电路如下:
:![](/images/img-107.png)

所以下拉输入的时候,输入高电平,下拉电阻不会把电平拉低。因为形成通路后,位置1电源就是VDD,高电平,不受下拉电阻的影响。

总结一下:
||引脚高电平|引脚低电平|引脚悬空(默认输入值)|
|------|------|------|------|
|输入上拉|1|0|1|
|输入下拉|1|0|0|

>弱上拉、弱下拉:MCU内部的上下拉电阻之所以被称为“弱”,是因为其阻值较⼤（通常在⼏⼗kΩ）,导致其提供的电流⾮常⼩,从⽽对外部信号的控制⼒⽐较“弱”。

## 3.水管模型
### 3.1 核心名词
把抽象的电学概念，翻译成具象的水管世界：

|电学概念|	水管模型对应物|	核心功能与特性
|------|------|------|
|VDD / VCC (电源正极)|	高压水塔 / 水泵出口|	能量的源头，提供高水压，负责把水推出去。
|VSS / GND (电源负极/地)|	地下排水总管 / 巨大的蓄水池|	系统的0米水位基准线。所有用过的水最终都要排回这里。
|负载 (芯片/电阻/电机)|	你家里的水龙头 / 用水管|	做功的地方。水流经过这里，推动水轮机（做电功）后压力降低。
|MOS管 (晶体管)|	电子水龙头 / 阀门|	控制水流开关。通电时阀门全开（导通），断电时阀门紧闭（截止）。
|电流|	水流|	从水塔出发，经过负载，最终流回蓄水池的物理闭环。
|电压|	水压 (水位差)|	推动水流的力量。两点间水位差越大，水压（电压）越大。
|PCB铜皮 (电源/地平面)|	总供水管 / 总排水管	|粗大的管道。电阻小，通水能力极强，能瞬间保证水压稳定。
|细导线|	细水管 / 细线|	阻力大，容易“堵塞”（压降大），受干扰严重（劣质设计）。
### 3.2 两个"0V"的终极真相:地的双重身份
这是最容易混淆的地方，水管模型完美解释了它：

角色一：静态的“海平面”（基准线）
状态： 你家没开水龙头。蓄水池里的水安静地待着。
作用： 它是测量高度的基准。我们说一个点水压是3.3个大气压，就是比这个0米蓄水池高3.3个大气压。
电流： 没有流动。

角色二：动态的“回家之路”（回路的一半）
状态： 你打开了水龙头，水从水塔流出，经过你家，最后掉进排水管流回蓄水池。
作用： 它是水流回家的必经之路。没有这条排水管，水龙头里的水就流不出来。
电流： 正在流动。
核心法则： 同一根GND线，没电流流过时是“基准线”；有电流流过时，就是“回流路径”。
### 3.3 一个完整世界的循环(打破认知误区)
误区： 很多新手认为电流是从 VDD 出发，经过芯片，到 VSS 就结束了（像一个“V”字形）。
真相（水管模型）： 这是一个完整的“O”字形闭环。

|水流（电流）的完整物理旅程：|
|------|
|起点： 水塔（电池正极） —— 能量从这里发出。|
传输段： 总供水管（PCB VDD铜皮） —— 把高压水送到你家门口。
入水口： 你家入水口（芯片VDD引脚） —— 水进入你的家庭系统。
做功段： 家里水管（芯片内部负载） —— 水推动水龙头做工，压力下降。
出水口： 你家排水口（芯片VSS引脚） —— 做完功的水流出。
回收段： 总排水管（PCB GND铜皮） —— 水流汇入巨大的地下管网。
终点（同时也是起点）： 蓄水池（电池负极） —— 水回归原位，准备再次被抽回水塔。

一句话总结： 你画电路图时看到的“从VDD到VSS”，只是你家里那段管道的局部特写。真正的世界是一个庞大的完整闭环。
### 3.4 极端场景
利用水管模型，我们可以瞬间秒懂复杂电路现象：

场景A：VSS连GND为什么是0V？
就像你把一根空水管直接插进地下蓄水池。排水管里没有水压，自然就是0米水位。这是个完美的参考点。

场景B：PMOS和NMOS同时导通（致命短路）
相当于你把家里的进水管（高压水塔）和排水管（0米蓄水池）用一个极粗的、敞开的管道直接连起来了！水（电流）会毫无阻力地瞬间狂喷，水压瞬间崩塌（电压拉低），你家水管和机器（芯片）瞬间爆裂（烧毁）。
:![](/images/img-112.png)

场景C：开漏输出0，接上拉电阻
相当于你在进水管和排水管之间，接了一根非常细长的管子（大电阻），并且阀门（MOS管）打开。此时会有一小股涓涓细流（微小电流）从高压水塔缓慢漏到蓄水池。这股细流确定了排水管的压力是0V（低电平信号）。
### 3.5 GND长什么样?是"地毯"不是"项链"
劣质设计（项链模型）： 把所有房间的下水道，都连到一根很细的铁丝上，最后再排到蓄水池。如果几家人同时洗澡，这根细线立刻堵死（地弹、干扰）。

优质设计（地毯/地平面模型）： PCB上有一整层巨大的实心铜皮（像一块大铁板），直接铺在蓄水池上方。所有房间的下水管道，直接焊穿捅进这块铁板里，瞬间被巨大的蓄水池吸收。

结论： 你在原理图上看到的那个小小的“GND”符号，其实代表的是PCB上那整片巨大的铜皮地平面。

“每当你看不懂电路时，就把正极想成‘水塔’，负极想成‘大海’，电流就是水流。只要是电流，就必定是从水塔出发，经过负载，最后流回大海，形成一个不漏水的闭合成环。”
