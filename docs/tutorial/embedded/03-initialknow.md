# STM32F103ZET6
## 1.初始STM32F103ZET6
ST32即ST意法半导体公司开发的32位微控制器。

STM32F103ZET6即STM32F103的一个系列(STM32F103也是STM32的一个系列)。STM32F103系列是意法半导体的经典Cortex-M3内核MCU,高性价比、丰富外设,入门首选。ZET6是产品规格(针脚数/Flash容量/封装格式/工作环境)

STM32能做什么? 
STM32作为微控制器,自带了各种常用通信接口,比如USART、I2C、SPI等,可外接很多的传感器,可以控制很多的设备。比如在现实生活中,我们接触到的很多电器产品都有STM32的身影,比如智能手环、微型四轴飞行器、平衡车、移动POS机、智能电饭锅、3D打印机等等。

## 2.STM32F103ZET6引脚
像不同种类的MCU引脚是要在数据手册中看到。
![](/images/img-40.png)

引脚内部有对应的编号,但我们不用这个,我们用的是外部的名字(这个名字是描述引脚的功能)。其实说人话就是我们后面编写代码对这些引脚读写电信号,从而实现对外设的控制。这里既可以感受到微控制器的作用。

BOOT0:启动模式选择(单片机复位后,从谁提供的软件指令开始执行)
![](/images/img-44.png)

|BOOT0状态|作⽤|通俗理解|
|------|------|------|
|接GND|运⾏已烧录的程序|正常开机，执⾏你烧录的程序。99%场景下，都是这个。|
|接3.3V| 进⼊Bootloader，等待串⼝下载|进⼊刷机模式，执⾏意法提供的Bootloader。当未来⽆法烧录的时候，可以使⽤它强制烧写|

• OSC_IN / OSC_OUT :外部⾼精度时钟输⼊/输出引脚

• VREF 系列:⾼精度 ADC/DAC 的参考电压

• NC :No Connect ,未进⾏任何连接的引脚
#### 2.1.1 NRST复位引脚 VS BOOT0
NRST复位引脚:不改变启动源,只是从头开始运行指定的软件.
BOOT0启动模式选择:不触发重启,只决定重启后运行谁的问题。

使用Windows来类比:NRST决定是否重启,BOOT0决定开机从哪个盘启动。
### 2.2 普通引脚
MCU的编程接口:功能由我们自己的软件来决定。可动态配置为输入(读取按键)、输出(控制LED)或复用UART、I2C、SPI、PWM、ADC等特定外设功能。
PA~PG(0~15个)7*16=112个普通引脚
## 3.仿真器
仿真器作用:需要把编写好的代码,烧录到STM32开发板中。

仿真器是⼀种硬件调试⼯具.通过仿真器连接开发板和PC,从⽽通过PC上的开发⼯具,对MCU进⾏烧录,调试选择用的DAPLink。
烧录本质就是把编译⽣成的可执⾏程序通过仿真器,写⼊STM32⽚上Flash中。
### 3.1 仿真器连接
![](/images/img-45.png)
使用杜邦线连接仿真器和开发板

仿真器1号引脚(RST)连接到开发板的RST引脚。仿真器5/6号引脚(GND)连接到开发板的GND引脚。仿真器4号引脚(SWDIO)连接到开发板的DIO引脚。仿真器2号引脚(SWCLK)连接到开发板的CLK引脚。
![](/images/img-46.png)

什么是SWDIO?
SWDIO是串行调试接口的双向数据线,用于在调试器和目标芯片之间传输命令与数据。

什么是CLK?
SWCLK是由调试器提供给目标芯片的时钟信号,用来同步SWDIO上的数据传输。

仿真器本身就是一个单片机系统
