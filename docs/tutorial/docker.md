# 认识docker
## 安装SpeedNote
你今天实际就干了这三件事
第一件： 你的 SpeedyNote 源码在 F:\speednote\SpeedyNote，这是盖楼用的建筑材料（钢筋水泥）
第二件： Docker 就是一个自动化的包工队。你让它 build，它自己下载了搅拌机、电钻、焊机（也就是 SDK、NDK、Qt），然后把你的钢筋水泥搭起来
第三件： 包工队干完了活，给了你一个成品——SpeedyNote.apk，你拿到平板上一装，完事。
整个过程中，你不需要知道包工队内部怎么运作，你只需要知道：
1. 给它材料（源码文件夹）
2. 它自动干活（敲两条命令等一会儿）
3. 它给你成品（APK）
那你没 Docker 会怎样？
你得自己当包工头：
- 自己去网上下载搅拌机、电钻（Android SDK、NDK、Qt）
- 自己研究这些工具怎么装、版本要匹配哪个
- 装好了还要配线路（环境变量）
- 出了问题只能自己百度修
Docker 帮你把**"自己当包工头"的过程变成了"雇一个成熟的公司干"**。
你不是问了这个问题吗？
这下载成功了，安装到平板上了，不卡了
对，你后来直接在 SpeedyNote 官网下载 APK 装上了，那个也是别人用类似 Docker 的环境编好的。
所以 Docker 做的事情很简单：让你自己也能生成那个 APK，而不是只能等官网发。
