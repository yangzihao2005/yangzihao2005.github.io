# 嵌入式入门总纲
宏观认识**嵌入式行业发展**

## 1. 计算机是怎么走到今天的？
### 1.1 计算机的发展
计算机的诞生来自**战争需求**（算炮弹轨迹、破密码）→ 国家愿意砸钱搞研发 → 但光靠政府投钱不可持续 → 所以必须**军转民**，让技术流入市场，形成商业循环。

在这个循环中，芯片厂商为了抢占市场，必须不断让芯片**更快、更便宜**——这就是**摩尔定律**（每18~24个月晶体管翻倍）的驱动力来源。摩尔定律不是政府规划的，是商业竞争逼出来的。

![](/images/img-24.png)

芯片便宜了、变小了，就能塞进各种各样的民用设备里——这就分出了三条路：
| 计算机类型 | 使命 | 典型产品 |
|:---|:---|:---|
| 通用计算机 | 什么都能干 | PC、笔记本 |
| 超级计算机 | 解决人类级难题 | 天气预报、密码破译 |
| 专用计算机（嵌入式） | 低成本解决某一件事 | 空调、无人机、洗衣机 |

> **我的理解**：嵌入式的核心就是“够用就好，不多花一分钱”。
### 1.2 芯片世界的两条路线
在民用计算机的发展中，芯片公司走出了两条完全不同的路线：

|  | **Intel 路线** | **ARM 路线** |
|:---|:---|:---|
| **核心理念** | 自己设计、自己制造（IDM模式），追求极致性能 | 只卖设计图纸（IP授权），追求极致低功耗 |
| **统治领域** | PC、服务器 | 手机、嵌入式设备 |
| **代表产品** | Intel Core i5/i7/i9、Xeon | 高通骁龙、苹果A/M系列、STM32 |

这两条路线在各自领域称王，井水不犯河水。但从2020年苹果Mac转向ARM架构开始，ARM正在用“低功耗+高性能”的组合拳，逐步入侵Intel的PC大本营。

> **我的理解**：Intel造的是“性能怪兽”，ARM造的是“省电大师”。以前各管各的，现在ARM开始跟Intel抢饭碗了。

## 2. 两个必须分清的概念

很多人把“单片机”和“嵌入式系统”混着叫，但它们在技术上是两个层面的东西：

| 概念 | 是什么 | 类比 |
|:---|:---|:---|
| **单片机 (MCU)** | 一颗芯片，集成了CPU、内存、外设接口 | **人的大脑** |
| **嵌入式系统** | 完整的专用计算机，包含MCU+电路+传感器+软件 | **整个人**（大脑+四肢+感官） |

> **我的理解**：单片机是“硬件核心”，嵌入式系统是“完整产品”。你买到的是一台空调（嵌入式系统），而不是一颗STM32芯片（单片机）。

嵌入式世界按核心处理器分为 MCU、应用处理器、FPGA、DSP 四大方向。实际上稍微复杂的嵌入式系统都不是由单一的嵌入式技术方向实现的，通常会将不同方向技术融合使用。

| MCU | 应用处理器 | FPGA |DSP|
|------|------|------|------|
| 集成 CPU、内存和外设的单芯片微型计算机，主打低功耗、低成本和实时控制，是入门嵌入式最主流的方向| 追求高算力，运行 Linux/Android 等复杂操作系统，面向手机、智能座舱等多媒体终端 | 是一块可重构的硬件“白纸”，用代码定义电路逻辑，追求极致并行和纳秒级响应 |专为数字信号处理算法优化，擅长音频、通信等高速数值计算。|

## 3. 谁在造芯片？谁在用芯片？

### 3.1 三条产业链分工

| 层级 | 代表公司 | 核心产出 | 一句话说清它在干嘛 |
|:---|:---|:---|:---|
| **IP授权层** | ARM | CPU设计图纸（IP核） | 不造芯片，靠“卖设计方案”赚钱，谁都能买 |
| **芯片层** | 意法半导体、高通、联发科 | SoC芯片/MCU芯片 | 买ARM图纸，加上自己的外设，找工厂代工，做出芯片卖掉 |
| **产品层** | 苹果、小米、大疆 | 手机、无人机、平衡车 | 买芯片，设计电路板，组装成产品卖给消费者 |

### 3.2 一个关键概念：SoC 和 CPU 什么关系？

> **SoC = 一整栋功能完整的大楼**（里面包含了CPU、GPU、内存控制器、WiFi模块等）
> **CPU = 这栋大楼里的“市政府”**（只负责决策，不管其他部门）

![](/images/img-25.png)

> **我的理解**：手机里不可能像电脑那样塞一块独立CPU再加独立显卡，空间不够、电池受不了。所以把所有核心部件集成到一颗芯片上——这就是SoC。而我们的STM32 MCU，本质上就是一颗“简化版”的SoC，专为控制而生。

## 4. ARM 的版图扩张与意法半导体
### 4.1 ARM 占领了哪些领域？
ARM 从手机起家，现在已经扩张到四个关键领域：

| 领域 | 说明 | 代表公司/产品 |
|:---|:---|:---|
| **移动计算（基本盘）** | 手机、平板，ARM 的绝对统治区 | 苹果A系列、高通骁龙、联发科 |
| **汽车电子（新增长极）** | 智能汽车 = “轮子上的超级计算机” | 特斯拉、蔚来、比亚迪的车机芯片 |
| **服务器与数据中心** | 传统 Intel 的领地，ARM 靠高能效比打入 | 华为鲲鹏、亚马逊 Graviton |
| **AIoT/物联网（广阔蓝海）** | 从微小传感器到复杂工业控制器 | STM32、瑞芯微、ESP32 |

### 4.2 ARM 为什么能扩张？

因为不同领域对芯片的要求完全不同：

| 应用场景 | 核心要求 |
|:---|:---|
| 智能手机 | 性能与功耗的平衡 |
| 汽车刹车系统 | 高实时性（响应必须快） |
| 智能手表 | 超低功耗（续航要久） |
| 服务器 | 高能效比（省电省钱） |

为了满足这些不同需求，ARM 设计了不同定位的 CPU 核心，**Cortex 系列**就是这些核心的统称：

| CPU系列 | 定位 | 典型场景 |
|:---|:---|:---|
| **Cortex-A** | 高性能（Application） | 手机、平板、服务器、智能座舱 |
| **Cortex-R** | 高实时性（Real-time） | 汽车刹车、硬盘控制器 |
| **Cortex-M** | 低功耗微控制器（Microcontroller） | IoT传感器、可穿戴设备、工业控制 |
| **Cortex-X** | 极致性能（Extreme） | 旗舰手机超大核 |

> **我的理解**：A 系列是“大力士”，R 系列是“快枪手”，M 系列是“省钱小能手”——不同活派不同的人干。

### 4.3 意法半导体（ST）：把 ARM 图纸变成芯片的公司

ARM 只卖图纸（IP 核），不造芯片。那么问题来了：如果我只是一个做智能家居的小公司，想用 ARM 核心做产品，难道要自己去买 ARM 图纸、设计芯片、找工厂代工吗？

——显然不现实。这太贵、太复杂了。

于是，**意法半导体（STMicroelectronics，简称 ST）** 这类公司站出来解决了这个问题：

> **意法半导体的角色**：从 ARM 买 Cortex-M 核心授权，加上自己的外设设计和存储器，封装成一颗即拿即用的完整芯片——这就是 **MCU**。

| | 说明 |
|:---|:---|
| **公司全称** | STMicroelectronics（意法半导体） |
| **名字由来** | 由意大利 SGS 和法国 Thomson 两家半导体公司合并而成 |
| **核心产品** | STM32 系列 MCU |
| **产品命名** | ST（意法半导体）+ M（微控制器）+ 32（32位） |
| **行业地位** | 事实上的行业基准：做得早、市占率高、生态最完整 |

>意法全球官⽹：www.st.com  ; 中⽂官⽹：www.st.com.cn

### 4.4 详细介绍MCU
MCU即单片机,是把处理器(CPU核心)、存储器(Flash、RAM)、以及各种输入输出接口(定时器、ADC、串口等)集成在一起，形成一个芯片。
![](/images/img-26.png)
>我的理解:MCU就是一台被集成在同一芯片上的微型计算机了。

|手机处理器SOC|MCU|
|------|------|
|解决算力问题(通用)|解决控制问题(专用)|

MCU就是一种专用的,为了省钱来解决控制问题。花小钱办大事。
如果用手机的SoC来用于嵌入式,有点杀鸡用牛刀的感觉,且实时性可能受影响。

## 5.MCU选型
### 5.1 为什么我们选 STM32？
很多厂商都生产MCU，其中意法半导体生成的MCU系列叫做STM32.

1. **生态最完整**：STM32 的开发生态（文档、教程、开源项目、工具链）是所有 MCU 中最庞大的，学习门槛最低。
2. **行业基准地位**：大量工程师的学习路径都是从 STM32 开始的，大量公司的新产品选型也优先考虑 STM32。
3. **后来者兼容**：国内厂商（如兆易创新 GD32）为了降低用户迁移成本，会主动做“引脚兼容、寄存器兼容”，学会 STM32 就等于学会了半个嵌入式行业。

## 7.STM32文档以及社区生态
### 7.1 STM32文档
该行业流程不同分工,不同分工导致专业知识壁垒,但是每个分工会整理出文档,阅读文档可以快速入门这个知识。

### 7.2 社区生态
MCU 的裸机开发:：在没有任何操作系统（如FreeRTOS、Linux）⽀持的情况下，直接在微控制器（MCU）上编写程序。
写代码要在MCU内部的Cortex-M3 CPU内核上运行.开发语言是C语言,而且需要编译器把C语言编译成二进制,在MCU内部运行。
